01.웹 표준 (Web Standards)

누구나, 어떤 환경에서든 웹 사이트에 접속한 사용자가 동일한 결과를 볼 수 있도록, 브라우저 종류나 버전에 상관 없이 호환이 가능하도록 제시된 기준을 말합니다.

1W3C validator

'HTML 유효성 검사'를 말하며, 검사를 통해 마크업에 대한 문법적 오류나 경고를 확인할 수 있습니다.
이는 웹 표준을 준수하는 HTML 문서작성에 가장 기본이 되는 사항이며, 마크업 중간 또는 CSS에 들어가기 전에 검사합니다.

W3C - 마크업 유효성 검사 바로가기   ☛ https://validator.w3.org/
W3C - CSS 유효성 검사 바로가기   ☛ https://jigsaw.w3.org/css-validator/
참고하기

1W3C(World Wide Web Consortium)

월드 와이드 웹을 위한 표준을 개발하고 장려하는 국제 협회를 말합니다.

2마크업

표시를 하여 문서를 구조화한다는 뜻이며, 태그를 포함하는 ' 요소(element)들로 코딩하는 것 '을 의미합니다.

2웹 표준을 위한 작업의 예

HTML / CSS 분리 HTML 문서에 CSS 파일을 링크로 연결합니다.
시멘틱 마크업 의미에 맞는 태그를 사용하여 논리적인 순서에 따라 문서를 작성합니다.
id / class 사용 특정 요소에 이름을 지정하여 표현 및 동작을 제어합니다.

2웹 표준의 장점

웹 표준의 장점
No 장점 설명
1 수정 및 관리 용이 콘텐츠의 문법적 구조화를 통해 제어하므로, 페이지 제작과 수정 및 관리가 쉽습니다.
2 호환성 및 접근성 향상 다양한 브라우저, 모바일, PDA, 장애인 지원용 프로그램 등에서 정상적인 작동이 가능하므로, 어떤 사용자라도 접근하여 이용할 수 있습니다.
3 검색 엔진 최적화 (SEO) 구조화된 웹 페이지는 검색 엔진에 효과적으로 노출됩니다.

02.웹 접근성 (Web Accessibility)

장애인, 노약자 등의 어떤 사용자가 어느 기술환경에서도 신체적 차이나 지식의 정도 등에 의한 불편함 없이,
웹 사이트에서 제공하는 모든 정보에 접근할 수 있도록 보장하는 것을 말합니다.

참고하기

1웹 접근성 도구의 예

스크린 리더기 (음성 낭독 프로그램) HTML 문서의 텍스트를 읽어줍니다.
화면 확대 도구 화면의 배율을 조절해줍니다.
음성 인식 도구 사용자의 음성을 텍스트로 입력합니다.
키보드 오버레이 화면에 가상의 키보드를 띄워줍니다.

2웹 접근성을 위한 작업

로그인, 보안, 구간 암호화, 공인 인증, 영상, 멀티 미디어, 파일 송수신 등에서 사용자의 접근성이 고려되었는지를 점검합니다.

CSS에서 사이즈 단위를 px 등으로 통일하는 것이 음성 인식에 편리합니다.

1웹 접근성을 위한 태그의 속성

1< html > 의 lang 속성

스크린 리더기는 lang 속성에서 설정한 언어에 따라 HTML 문서의 텍스트를 읽어줍니다.

<html lang = "ko">          // 언어를 한국어로 설정 : 숫자를 " 일, 이, 삼 " 으로 읽음
2< img > 의 alt 속성

스크린 리더기는 이미지를 읽을 수 없으므로, < img > 의 alt 속성 값에 대체 문자를 기재하여 이미지에 대한 정보를 제공합니다.
이미지에 대한 정보를 제공할 필요가 없을 경우에는 alt 속성만 정의해도 됩니다. 

<img src="assets/img/복숭아_단면.png" alt="씨앗이 보이도록 반으로 자른 복숭아">
<img src="assets/img/무화과_단면.png" alt>
3< a > 의 target, title 속성

< a > 의 target 속성은 링크로 연결된 HTML 문서를 어느 창의 위치에서 열어줄 것인지 정하는 속성입니다.
속성 값 중에서 가장 많이 사용하는 target="_blank"는 스크린 리더기에서 링크된 문서가 '새 창'에서 열린다는 것을 미리 알려주는 역할도 합니다.
Javascript 로 새 창을 열어주는 경우에는 title 속성으로 새 창으로 열린다는 것을 알려주어야 합니다. 

<a href="https://oranssy.tistory.com/" target="_blank">깜냥깜냥 코딩 혜윰</a>
<a href="#" onclick="window.open('https://oranssy.tistory.com/');return false" title="새 창">깜냥깜냥 코딩 혜윰</a>
참고하기

1target 속성 값의 종류

_self 현재 창에 링크를 열어줍니다. (기본값, 생략 가능)
_blank 새로운 창 또는 탭에서 링크를 열어줍니다.
_parent 현재 창의 부모 창에서 링크를 열어줍니다.
_top 현재 창의 최상위 창에서 링크를 열어줍니다.

4< table > 의 < caption >

스크린 리더기는 < caption > 에 기재한 표의 제목을 읽어주어, 표가 어떤 내용을 담고 있는지 알려줍니다.
caption 태그는 < table > 의 바로 밑 줄에 입력합니다.

참고하기

1< caption > 텍스트 숨기기

요소 내용의 텍스트를 CSS로 숨겨야 하는 경우가 있습니다.
이 때 / display: none; / visibility: hidden; / width: 0; / height: 0; / font-size:0; / 등을 사용하면
화면에 없는 영역으로 넘어가게 되어 설명 내용이 보이지 않을 뿐만 아니라, 스크린 리더기에서도 읽을 수 없기 때문에 사용해서는 안 됩니다.

2웹 접근성을 고려한 텍스트 숨기기 - 네이버의 blind 효과

.blind {
    position: absolute;
    clip: rect(0 0 0 0);
    width: 1px;
    height: 1px;
    margin: -1px;
    overflow: hidden;
}
5< table > 의 scope 속성

scope 속성은 제목 셀에 내용 셀과의 관계를 나타내는 것으로, 스크린 리더기가 셀을 읽는 순서를 알려주는 역할을 합니다.
행 또는 열의 제목 셀 < th >에 정의합니다.

참고하기

1행과 열

행 Row, 가로
열 Column, 세로

scope 속성 설정해보기
6< form > 의 < label >

스크린 리더기는 < label > 에 기재한 사용자 입력 태그의 제목을 읽어주어, 어떤 값을 입력하고 선택해야 하는지 알려줍니다.
< label > 태그의 내용이 사용자 입력 태그의 제목이라는 것을 알리기 위해서는
< label > 태그의 for 속성 값과 사용자 입력 태그의 id 속성 값이 일치해야 합니다. 

<label for="pw">비밀번호</label><input type="password" id="pw">

2한국형 웹 콘텐츠 접근성 지침 2.2

이 표준은 방송통신표준화법 관련 규정에 따라 방송통신표준심의회 심의를 거쳐 개정한 방송통신 표준 입니다.
장애인, 고령자와 같은 사람들이 비장애인, 젊은이와 동등하게 웹에 접근할 수 있도록, 웹 콘텐츠를 제작할 때 준수해야 하는 여러 가지 지침들을 기술하고 있습니다.

세부내용 보기
1인식의 용이성 (perceivable)

사용자가 장애 유무 등에 관계없이 웹사이트에서 제공하는 모든 콘텐츠를 동등하게 인식할 수 있어야 합니다.

인식의 용이성에 대한 세부 지침내용
지침 평가 지표
5.1. 대체 텍스트 (적절한 대체 텍스트 제공) 텍스트 아닌 콘텐츠는 그 의미나 용도를 인식할 수 있도록 대체 텍스트를 제공해야 한다.
5.2. 멀티미디어 대체수단 (자막 제공) 멀티미디어 콘텐츠에는 자막, 대본 또는 수어를 제공해야 한다.
5.3. 적응성 (표의 구성) 표는 이해하기 쉽게 구성해야 한다.
(콘텐츠의 선형구조) 콘텐츠는 논리적인 순서로 제공해야 한다.
(명확한 지시사항 제공) 지시사항은 모양, 크기, 위치, 방향, 색, 소리 등에 관계없이 인식될 수 있어야 한다.
5.4. 명료성 (색에 무관한 콘텐츠 인식) 콘텐츠는 색에 관계없이 인식될 수 있어야 한다.
(자동 재생 금지) 자동으로 소리가 재생되지 않아야 한다.
(텍스트 콘텐츠의 명도 대비) 텍스트 콘텐츠와 배경 간의 명도 대비는 4.5 대 1 이상이어야 한다.
(콘텐츠 간의 구분) 이웃한 콘텐츠는 구별될 수 있어야 한다.
2운용의 용이성 (operable)

사용자가 장애 유무 등에 관계없이 웹사이트에서 제공하는 모든 기능을 운용할 수 있어야 합니다.

참고하기

1운용 (運用)

도구의 기능과 성능을 적절하게 사용하는 것을 의미합니다.

운용의 용이성에 대한 세부 지침내용
지침 평가 지표
6.1. 입력장치 접근성 (키보드 사용 보장) 모든 기능은 키보드만으로도 사용할 수 있어야 한다.
(초점 이동과 표시) 키보드에 의한 초점은 논리적으로 이동해야 하며, 시각적으로 구별할 수 있어야 한다.
(조작 가능) 사용자 입력 및 콘트롤은 조작 가능하도록 제공되어야 한다.
(문자 단축키) 문자 단축키는 오동작으로 인한 오류를 방지하여야 한다.
6.2. 충분한 시간 제공 (응답시간 조절) 시간제한이 있는 콘텐츠는 응답시간을 조절할 수 있어야 한다.
(정지 기능 제공) 자동으로 변경되는 콘텐츠는 움직임을 제어할 수 있어야 한다.
6.3. 광과민성 발작 예방 (깜빡임과 번쩍임 사용 제한) 초당 3~50회 주기로 깜빡이거나 번쩍이는 콘텐츠를 제공하지 않아야 한다.
6.4 쉬운 내비게이션 (반복 영역 건너뛰기) 콘텐츠의 반복되는 영역은 건너뛸 수 있어야 한다.
(제목 제공) 페이지, 프레임, 콘텐츠 블록에는 적절한 제목을 제공해야 한다.
(적절한 링크 텍스트) 링크 텍스트는 용도나 목적을 이해할 수 있도록 제공해야 한다.
(고정된 참조 위치 정보) 전자출판문서 형식의 웹 페이지는 각 페이지로 이동할 수 있는 기능이 있어야 하고, 서식이나 플랫폼에 상관없이 참조 위치 정보를 일관되게 제공ㆍ유지해야 한다.
6.5. 입력 방식 (단일 포인터 입력 지원) 다중 포인터 또는 경로기반 동작을 통한 입력은 단일 포인터 입력으로도 조작할 수 있어야 한다.
(포인터 입력 취소) 단일 포인터 입력으로 실행되는 기능은 취소할 수 있어야 한다.
(레이블과 네임) 텍스트 또는 텍스트 이미지가 포함된 레이블이 있는 사용자 인터페이스 구성요소는 네임에 시각적으로 표시되는 해당 텍스트를 포함해야 한다.
(동작기반 작동) 동작기반으로 작동하는 기능은 사용자 인터페이스 구성요소로 조작할 수 있고, 동작기반 기능을 비활성화할 수 있어야 한다.
3이해의 용이성 (understandable)

사용자가 장애 유무 등에 관계없이 웹 사이트에서 제공하는 콘텐츠를 이해할 수 있어야 합니다.

이해의 용이성에 대한 세부 지침내용
지침 평가 지표
7.1. 가독성 (기본 언어 표시) 주로 사용하는 언어를 명시해야 한다.
7.2 예측 가능성 (사용자 요구에 따른 실행) 사용자가 의도하지 않은 기능(새 창, 초점에 의한 맥락 변화 등)은 실행되지 않아야 한다.
(찾기 쉬운 도움 정보) 도움 정보가 제공되는 경우, 각 페이지에서 동일한 상대적인 순서로 접근할 수 있어야 한다.
7.3. 입력 도움 (오류 정정) 입력 오류를 정정할 수 있는 방법을 제공해야 한다.
(레이블 제공) 사용자 입력에는 대응하는 레이블을 제공해야 한다.
(접근 가능한 인증) 인증 과정은 인지 기능 테스트에만 의존해서는 안 된다.
(반복 입력 정보) 반복되는 입력 정보는 자동 입력 또는 선택 입력할 수 있어야 한다.
4견고성 (robust)

사용자가 기술에 영향을 받지 않고 콘텐츠를 이용할 수 있어야 합니다.

견고성에 대한 세부 지침내용
지침 평가 지표
8.1. 문법 준수 (마크업 오류 방지) 마크업 언어의 요소는 열고 닫음, 중첩 관계 및 속성 선언에 오류가 없어야 한다.
8.2. 웹 애플리케이션 접근성 (웹 애플리케이션 접근성 준수) 콘텐츠에 포함된 웹 애플리케이션은 접근성이 있어야 한다.

03.웹 호환성 (Cross Browsing)

운영체제나 브라우저 등에서 어느 한쪽으로 최적화되거나 종속되지 않고, 어떤 매체를 통해서든 동등한 서비스를 이용할 수 있도록 공통 요소를 사용하여 웹 페이지를 제작하는 것을 의미합니다.
쉽게 말해서 웹 표준 기술을 바탕으로 최대한 많은 종류의 웹 브라우저에서 정상적으로 작동하는 웹 페이지를 제작하는 것이라고 할 수 있습니다.
