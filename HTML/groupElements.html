01.시멘틱 마크업과 그룹화

시멘틱 마크업을 위해서는 같은 맥락의 요소들을 묶어서 논리적인 순서를 갖는 중첩 구조를 만드는 것이 중요합니다.
마크업에서 그룹화가 제대로 이루어져야 CSS 를 이용한 레이아웃 작업이 수월해집니다.

참고하기

1시멘틱 마크업

HTML 문서의 코드를 사람과 브라우저 모두가 이해할 수 있도록 의미에 맞는 요소를 사용하여 구조화하는 것을 말합니다.

2레이아웃 (Page Layout)

제한된 공간 안에서 구성 요소들을 보기 좋게 효과적으로 배치하는 것을 말합니다.

02.그룹화 요소의 종류

1< header >

문서 또는 특정 섹션의 머리글을 그룹화합니다.
제목, 로고, 네비게이션, 검색 폼, 작성자 이름 등을 포함할 수 있습니다.

2< nav >

HTML 문서 내의 메인 메뉴나 목차 등의 링크들을 그룹화합니다.
보통 목록 요소 <li> 를 포함합니다.

3< main >

<body> 의 주요 콘텐츠를 그룹화하며, 문서에서 한 번만 사용할 수 있습니다.
전반적으로 반복되는 내용을 포함해서는 안 되며, <header>, <nav>, <article>, <aside>, <footer> 등의 자식 요소로 존재할 수 없습니다.

4< section >

같은 맥락의 요소들을 주제별로 그룹화합니다.
적절한 의미를 갖는 컨테이너 요소가 없을 때 사용하며, 제목 요소를 포함해야 합니다.
<nav>, <article>, <aside>, <main> 등의 컨테이너 요소를 선택하는 것이 적절하다면, 해당 요소를 사용해야 합니다.

5< article >

HTML 문서 내에서 독립적으로 배포하거나 재사용할 수 있는 콘텐츠 내용을 그룹화합니다.
제목 요소를 포함해야 합니다.
게시물, 뉴스 기사, 블로그 포스팅 등의 내용을 다룹니다.

6< aside >

문서의 주요 콘텐츠와 직접적으로 관련이 없는 보조 내용을 그룹화합니다.
사이드 메뉴, 광고 등의 내용을 다룹니다.

7< div >

의미를 갖지 않으며, 그룹화 등의 문서의 영역을 나누는 블록 성질의 컨테이너를 만듭니다.
CSS를 설정하기 전에는 콘텐츠나 레이아웃에 어떤 영향도 주지 않습니다.

8< span >

의미를 갖지 않으며, 그룹화 또는 속성 값을 공유하는 인라인 성질의 컨테이너를 만듭니다.
CSS를 설정하기 전에는 콘텐츠나 레이아웃에 어떤 영향도 주지 않습니다.

9< footer >

문서나 특정 섹션의 바닥글을 그룹화합니다.
구획의 작성자, 저작권 정보, 연락처, 사이트맵, 관련 문서, top 버튼 등의 정보를 담습니다.

03.< section >, < article >, < div >, < span > 비교

시멘틱 마크업을 위해서는 같은 맥락의 요소들을 묶어서 논리적인 순서를 갖는 중첩 구조를 만드는 것이 중요합니다.
마크업에서 그룹화가 제대로 이루어져야 CSS 를 이용한 레이아웃 작업이 수월해집니다.

section, article, div, span 요소 비교
< section > < article > < div > < span >
의미 부분, 구획, 마디 기사, 논설, 논문 없 음
다루는 주제 하나의 일반적인 주제로 묶을 수 있는, 맥락이 같은 요소들 독립적으로 배포하거나 재사용할 수 있는 내용의 요소들
